# Anchor 正式蒸馏契约

> 这是正式管线的单一事实来源。旧的 c10、384+128、网页模板生成器和
> direct-API collector 仅是历史/合成证据，不能替代本文定义的正式路径。

## 1. 题库边界

- 只导入锁定 immutable revision 的官方 SWE-bench/SWE-smith **train** 分区；
  完整处理全部符合条件的 train 题，而不是只取 512 题。
- 当前已核验的正式源锁为 `SWE-bench/SWE-bench` revision
  `7074ef12ea2a6f70a228943c1336553333c22786`：train 共 19,008 行、
  19,008 个唯一 instance ID、35 个仓库；原始 parquet 为 106,492,326 bytes，
  本地 SHA-256 与官方 LFS 一致：
  `0ee19c80623ebc6eeef483b597dd38f27c1dda22054e00210976d315cea87a69`。
- 记录上游 revision、原始文件哈希、精确题数、仓库分布、过滤原因和派生清单。
- 验证集必须从 train 内用固定 seed 确定性划分；不得读取、翻译、路由、训练或
  输出官方 test/dev/heldout 题目正文。
- 每个公开数据文件小于 50 MiB；提交前执行密钥、heldout、许可证和大文件审计。

## 2. 五段依赖链

每道源题必须保留同一 `task_id` 并按依赖顺序生成：

1. `planner`：结合上下文形成严格 JSON 计划，明确要调用哪个构建专家、修改目标、
   验收条件、依赖和风险。
2. `tool_policy`：在构建前审查计划与拟调用工具，给出允许、修改后允许或阻断结论。
3. `builder`：在一次性沙箱内真实检出仓库、编辑、调用工具并运行构建/测试。
4. `reviewer`：对应领域审查代码、测试与差异；不合格时给出结构化修订请求并回到构建。
5. `security`：对最终产物做安全门控，输出可审计的放行/阻断及依据。

五段不能被生成为五个互不相关的回答。前序产物哈希、工具调用、工具结果、构建结果、
最终 diff/verdict 必须形成可核验链路。

## 3. 教师与 MAX 档位

- 正式教师为 `GLM-5.2` 和 `Kimi-K3`。
- **两者每一次正式请求均必须使用供应商定义的精确 `MAX` 推理档位。**
  `high`、默认值、缺失值或路由中被静默降级都视为配置错误，预检必须 fail-closed。
- GLM-5.2 作为广覆盖教师；Kimi-K3 同样参与正式题库，并优先承接前端、多模态和
  构建密集型题目。分配必须确定性、可复现、写入不含正文/密钥的路由清单。
- 系统不能写死供应商或模型。协议、Base URL、模型 ID、推理档位、路由和价格元数据
  均可由用户配置；支持用 URL + 进程内 key 拉取模型列表，拉取失败时允许手动指定或
  强制手动指定模型 ID。GLM-5.2 MAX 与 Kimi-K3 MAX 是本次正式 profile，不是产品上限。
- API key 只存在于父进程内存/环境变量并由子进程继承；退出进程或重启即清除。
  不得写入 YAML、Git、示例、日志或仓库内 secret store。

## 4. 训练样本内容

正式样本保留：模型可见输入、结构化决策/理由轨迹、严格阶段输出、工具调用参数、
真实工具结果、构建/测试证据以及最终 patch 或 verdict。不得用文字叙述伪造工具执行，
也不得虚构教师未返回的隐藏思维链。

当前版本不把 OpenCode 易变的 system/front prompt 放进训练样本；保留任务输入、
模型输出和真实执行轨迹，使模型学习协议和工作过程，同时避免绑定某个客户端提示词版本。

训练题库采用明确的 `real_sandbox_self_verified` 证据等级，不把它冒充官方
SWE-bench 判定。每道入选题必须由 supervisor HMAC 回执绑定：完整五阶段 artifact
谱系、精确 candidate task/work-order 分片哈希、隔离工具轨迹、终态且非平凡的零退出码
`anchor-validate compile|test` 结果、解析后的 validator JSON 与独立重算的终态、精确
final patch、不可变镜像 digest/ID、validator 源码哈希、仓库/base 身份以及成功清理。
消费端还会独立核验 publication inventory 与上述全部绑定；manifest 必须写入
`not_official_swebench_pass=true`。证据缺失的题继续排除并可重试；认证证据损坏或
被篡改时 fail-closed。封顶的 `stopped_checkpoint_resumable` 运行可以导出已认证成功
题，但在覆盖全题库前必须写入 `not_for_full_bank_completion_claim=true`。官方 heldout
评测保持独立，绝不进入训练。

## 5. 中英双语

- train 派生视图必须中英混合，并保留原文、翻译版本、来源和语义配对关系。
- 只翻译自然语言；代码、标识符、路径、命令、JSON key、工具参数/结果、哈希和许可证
  原文不得翻译。
- 翻译失败不得覆盖原样本；质量门需要检查结构、占位符、代码块和语义一致性。

## 6. 双魔改运行路径

- 魔改 OpenCode：负责 agent 会话、真实工具调用/回传、一次性 WSL2/Podman 沙箱、
  build/test、session/diff 导出与清理。
- 魔改 CC Switch：负责 provider/model 切换、路由、MAX 档位透传、价格表、token/成本、
  usage 和请求统计。
- 正式启动器若缺少任一侧的可核验 manifest、路由配置或沙箱预检，必须拒绝启动；
  不得静默回退到 direct-API synthetic collector。

## 7. 控制面板与运行策略

`http://127.0.0.1:8765/` 是标准操作入口，必须能配置 URL、进程内 key、模型 ID、
并发、重连时间和最大重试次数，并显示开始/停止/继续、速度、token、成本、进度、
前后端断连状态及重连原因。默认并发为 1 以便复现；通过预检后由用户显式调高，
正式产品不写死全局并发上限。

网络路径必须显式。国内资源和大体积下载（尤其 10 GiB 以上）默认绕过系统/虚拟网卡
代理并使用物理网卡默认路由；检测到代理或虚拟路由时，面板在下载前显示 direct/proxy、
预计大小和目标磁盘并允许用户覆盖，不能静默消耗代理流量。

`NO_PROXY=*` 或清空代理环境变量只代表“绕过环境代理”，不能在 TUN 默认路由优先时
冒充“物理网卡直连”。真正的 physical 模式必须对供应商主机使用可审计、可恢复的路由
固定或等价的接口绑定；需要管理员权限而无法完成时，应明确提示并 fail-closed，或由用户
显式选择继承当前路由。本轮配置和预检不得自动改动系统路由。

## 8. 发布边界

代码、配置、数据清单和合规的数据分片可提交并 push。未经用户新的明确指令，
不得创建 Git tag，也不得发布 GitHub Release 或其他版本包。

所有面向使用者的说明必须分别提供简体中文和英文文件，且命令、配置字段、证据边界、
故障排查和运行状态保持同步；不能只提供机器翻译摘要或让读者自行拼接多份文档。
